1. Core Data Structures & Schemas
To support the new systems, we need structured data definitions. All items (Pickaxes and Fortunes) should inherit from a common base or interface to allow for modular stat calculation.
1.1 Enums & Types
type Rarity = 'Common' | 'Improved' | 'Rare' | 'Epic' | 'Legendary';
type Planet = 'Earth' | 'Moon' | 'Mars' | 'Jupiter' | 'Titan' | 'Any';

interface PlayerStats {
 luck: number;        // e.g., 1.0 is 100% (base)
 lootFind: number;    // Integer value, e.g., 80
 wow: number;         // e.g., 1.0 is 100%
 critChance: number;  // e.g., 0.05 is 5%
 dpcMultiplier: number; // For Staff of Sundoge effects
 helperDpsMultiplier: number;
 rocketCostReduction: number;
}

1.2 Pickaxe Schema
We have simplified the model: DPC is Power. There is no hidden strength stat. The higher the DPC, the more damage you deal to rocks and the more coins you get.
interface Pickaxe {
 id: string; // Derived from folder name
 name: string;
 description: string;
 rarity: Rarity;
 planetOfOrigin: Planet;
 
 // Visual Assets
 idleSprite: string; // Path to idle.png
 activeSprite?: string; // Path to active.png (optional)
 
 // Base Stats (Generated on drop)
 // This value acts as the "Damage" against rocks AND the "Income" per click.
 baseDPC: number; 
 
 // Bonus Stats
 // Contains both Core Stats (affect gameplay) and Cosmetic Stats (affect UI only)
 // Also tracks the display indicator (+, +%, -) for the UI
 stats: Record<string, { value: number, indicator: '+' | '+%' | '-' }>; 
 
 // Special Flags
 isStaffOfSundoge?: boolean; // If true, applies Helper DPS boost instead of DPC
 specialInstructions?: string; // Any custom mechanical hooks from the txt file
}

1.3 Fortune Schema
Fortunes are passive items stored in a separate inventory.
interface Fortune {
 id: string;
 name: string;
 rarity: Rarity;
 description: string;
 stats: Partial<PlayerStats>; // Fortunes generally use core stats, but can be expanded to use the Pickaxe stats model if cosmetic fortunes exist.
}

2. The Pickaxe System & Data
2.1 Global Linear Progression (The "Always Better" Rule)
Unlike the original game which might use fixed stats per planet, CE requires a strict linear upgrade path.
* Logic: When a pickaxe drops, its stats are not pulled directly from a static file. Instead, they are calculated relative to the player's current best pickaxe.
* Formula: NewPickaxe.DPC = CurrentMaxDPC + UpgradeStep
* Planet Independence: If a player has a high-tier pickaxe from Titan, and then returns to Earth and finds a "Normal Pickaxe", that "Normal Pickaxe" will be generated with stats higher than the Titan pickaxe.
Generation Routine:
1. Get CurrentMaxDPC (DPC of the player's best owned pickaxe).
2. Identify the drop template (e.g., "Normal Pickaxe" from Earth pool).
3. Calculate NewDPC:
   * If pickaxe template specifically overrides DPC to 0 (e.g. Staff of Sundoge), NewDPC = 0.
   * Else, NewDPC = CurrentMaxDPC * 1.15 (15% upgrade).
2.2 Pickaxe Asset & Data Structure (Folder Based)
Instead of a single JSON registry, each pickaxe template is stored in its own dedicated folder. The game must read these directories to populate the drop pools.
Folder Structure Example:
/assets/pickaxes/earth_common_normal/
* idle.png (Required: The default sprite)
* active.png (Optional: The sprite shown when the pickaxe is used/clicks the rock)
* data.txt (Required: Contains the template metadata)
The data.txt File Parser:
Claude needs to implement a parser for these .txt files. Notice that no base numbers are provided for stats in the text file. The .txt file only dictates which stats the pickaxe can roll and how they are displayed. The actual numerical values for those stats are randomly generated upon drop based on the player's "Wow" stat.
Example data.txt content:
Name: Normal Pickaxe
Description: Normal, yet faithful.
Rarity: Common
Stats: +luck, +%lootFind, +dancingSkills, -sadness
Instructions: [Optional specific features or hooks for this pickaxe]

CRITICAL RULE - Stat Parsing & Indicators:
When parsing the Stats: line, Claude must observe the prefix on each stat (+, +%, -) and separate the core vs cosmetic stats.
1. Stat Indicators:
   * + (e.g., +dancingSkills): A standard additive stat. Generate a flat number. UI format: +45 Dancing Skills.
   * +% (e.g., +%lootFind): A percentage multiplier. Generate a decimal/percentage value. UI format: +15% Loot Find.
   * - (e.g., -sadness): A reduction stat. These are beneficial (reducing a bad thing). Generate and store as a positive number internally (the amount of reduction), but the UI must format it with a minus sign: -20 Sadness.
2. Core vs Cosmetic Stats:
Claude must cross-reference the parsed stat names (ignoring the prefix) with the PlayerStats interface (Section 1.1).
   * If a stat is listed in PlayerStats (e.g., luck, wow, critChance), it is a Core Stat and affects gameplay calculations.
   * If a stat is NOT listed in PlayerStats (e.g., dancingSkills, discipline, sadness, swag), it is a Cosmetic Stat. It has absolutely NO gameplay benefit. It should be given a randomly generated number and displayed in the UI purely for flavor/humor.
3. Mining Mechanics (Rock Hardness)
The core mining loop must account for Rock HP scaling and Pickaxe Power (DPC).
3.1 Rock HP Scaling Formula
Every time a rock is broken (rocksBroken counter increments), the next rock has more HP.
function getRockHP(baseHP, rocksBroken) {
 const growthFactor = 1.15; // 15% growth per rock (adjustable)
 return Math.floor(baseHP * Math.pow(growthFactor, rocksBroken));
}

3.2 Damage Calculation
DPC is Damage. The rock gets harder by having more HP, so higher DPC is required to break it quickly.
function calculateClickDamage(playerStats, equippedPickaxe) {
   let damage = (equippedPickaxe.baseDPC) * playerStats.dpcMultiplier;

   // Apply Critical Hit
   const isCrit = Math.random() < playerStats.critChance;
   if (isCrit) {
       damage *= 2.5;
   }

   return damage;
}

4. Loot Drops & The Fortune System
4.1 Drop Intervals
Loot is rolled when rock health crosses these thresholds (tracked in state):
   * 90% & 75%: Coins Only.
   * 50%, 25%, 0%: Chance for Dogebags (Pickaxes/Fortunes) or Coins.
4.2 Drop Logic with Stats
We need three distinct rolls influenced by player stats.
   1. Drop Trigger Roll: (Does a bag drop?)
   * Base Chance: 10% at relevant intervals.
   * Modifier: playerStats.luck.
   * Chance = Base * (1 + playerStats.luck)
   2. Rarity Roll: (Common vs Legendary?)
   * Modifier: playerStats.lootFind.
   * This logic uses a "Weight Reduction" system. Higher rarity has a high "Roll Threshold" (e.g., 1000). Loot Find reduces this threshold.
   * Formula: EffectiveThreshold = BaseThreshold / (1 + (LootFind / 100))
   3. Item Generation (The "Wow" Stat):
   * When an item is generated, its secondary stats (luck, wow, crit, and all cosmetic stats) are rolled within a range.
   * Modifier: playerStats.wow.
   * FinalSecondaryStat = BaseStat * (1 + Random(0, playerStats.wow))
   * Note: Primary stats (DPC) are determined by the Linear Progression System (Section 2.1), NOT the Wow stat.
4.3 Coin Drop Rebalancing (Power Based)
We are moving away from the "Current Balance" calculation to a "Power Based" calculation to prevent snowballing and ensure pickaxe upgrades feel impactful.
Coin Drop Formula (Standard Pile):
function calculateCoinDrop(equippedPickaxe, playerStats) {
   // Base value is derived directly from the Pickaxe's Dogecoin Per Click (DPC).
   // This ensures better pickaxes = bigger drops.
   let baseDrop = equippedPickaxe.baseDPC * 50; 

   // 'Loot Find' and 'Luck' act as multipliers for the quantity.
   // e.g., 20 LootFind = 1.2x coins.
   let multiplier = 1 + (playerStats.lootFind / 100) + (playerStats.luck / 10);

   return Math.floor(baseDrop * multiplier);
}

4.6 Dogebag Coin Rewards
If the Dogebag contains coins instead of an item, the amount is significantly higher than a standard pile.
   * Logic: BagCoinDrop = Random(1.5, 2.0) * calculateCoinDrop(...)
   * Result: A bag of coins is always worth 1.5x to 2x more than the maximum possible standard coin pile drop for the player's current stats.
